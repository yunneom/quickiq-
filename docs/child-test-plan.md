# 아동 IQ 테스트 — 사전 설계 문서 (Pre-Launch Plan)

> **상태**: DRAFT — 성인용 LS 승인 + 첫 매출 검증 후 착수
> **결정 보류 이유**: 법적·심사 리스크 폭증, MVP 단계 포커스 분산
> **이 문서의 목적**: 진행 결정 시 즉시 착수 가능한 설계 청사진

---

## 1. 왜 지금이 아닌가 (재확인)

| 리스크 | 영향 |
|---|---|
| 한국 의료법·표시광고법·아동복지법 | "아동 IQ 측정" 마케팅이 사교육·의료 광고 규제 영역에 진입 |
| LS 추가 KYC | 미성년자 대상 상품은 별도 검증 → 성인용 store까지 보수적 검토 |
| COPPA (US) | 13세 미만 데이터 수집 시 부모 동의 인프라 필수 |
| Meta 광고 정책 | 미성년자 타겟·정신검사 광고는 거절 빈도 높음 |
| MVP 일정 분산 | 성인용 첫 매출도 미발생 → 검증 전 segment 추가 위험 |

**Go 조건 (이게 다 충족되어야 시작)**:
- [ ] 성인용 LS 승인 완료
- [ ] 30일 누적 매출 200만원+ (PMF 검증)
- [ ] 법무 검토 비용 회수 가능한 수익 구조
- [ ] CPA 확보 (CAC < LTV 검증)

## 2. 타겟 연령대 설계

| 세그먼트 | 연령 | 페르소나 | 결제 주체 |
|---|---|---|---|
| Junior | 만 6–9세 (초1–초3) | 부모가 응시 시킴, 부모 결제 | 부모 |
| Tween | 만 10–13세 (초4–중1) | 본인 호기심도 있음, 부모 결제 | 부모 |
| Teen | 만 14–18세 (중2–고3) | 본인 응시·공유, 부모/본인 결제 | 부모 또는 본인 |

**MVP 권장**: Tween (10–13세) 단일 세그먼트로 시작. 이유:
- Junior는 텍스트 읽기 어려움 → 이미지/음성 UX 추가 비용 큼
- Teen은 성인용으로 흡수 가능 (광고 카피만 차별화)
- Tween은 ① 부모 결제 의사 가장 강하고 ② 본인이 흥미 가지고 ③ 텍스트 응시 가능

## 3. 콘텐츠 설계 — 성인용 대비 차이

### 3.1 문항 차이
| 차원 | 성인용 (현재) | Tween용 |
|---|---|---|
| 어휘 난이도 | 평균 어휘 수준 | 초등 4–6학년 교과 어휘 한정 |
| 문항 길이 | 1–3문장 | 1문장 (이미지로 보조) |
| 영역 비율 | verbal/numerical/spatial/logical 균등 | spatial/logical 비중 ↑ (어휘 의존도 ↓) |
| 응시 시간 | 7분 | 5분 (집중력 한계 고려) |
| 문항 수 | 30 | 20 |
| 이미지 의존 | 낮음 | **높음** (도형·패턴 그림 다수 필요) |

### 3.2 스코어링 차이
- 성인용: 평균 100·SD 15 정규분포 매핑 (성인 인구 기준)
- 아동용: **연령별 평균값 보정 필수** — 만 10세와 만 13세를 같은 척도로 비교하면 안 됨
- 결과 표기: "추정 IQ" 대신 "**또래 비교 점수**" (예: "또래 100명 중 상위 25%") — 어린이에게 절대 수치 IQ 라벨 부여 지양

### 3.3 리포트 차이 (PDF)
- 성인용: 직업/학습 시사점
- Tween용: **부모용 리포트** + **자녀용 격려 메시지** 분리
  - 부모: 강점 영역, 학습 보조 팁, "이건 진단이 아닙니다" 강한 disclaimer
  - 자녀: 친근한 캐릭터/일러스트, 격려 중심 (점수보다 영역별 강점)

## 4. 법적·기술적 요구사항

### 4.1 한국
- **개인정보보호법**: 만 14세 미만은 법정대리인(부모) 동의 필수
- **표시광고법**: "우리 아이 IQ 측정" 같은 의료성·진단성 표현 금지
- **아동·청소년 보호 자율 규제**: 광고에서 점수 등급으로 자극적 카피 금지
- **사교육 마케팅 윤리**: "이거 안 하면 뒤처져요" 류 FOMO 카피 절대 금지

### 4.2 미국 (영문 버전 운영 시)
- **COPPA**: 13세 미만은 부모 동의 + 데이터 수집 최소화 + 30일 내 삭제
- **FTC 광고 규정**: "Diagnose / Evaluate / Measure IQ" 단어 피하기, "Fun cognitive activity" 표현 사용

### 4.3 데이터 처리
- 응시자 연령은 수집하되 식별 정보(이름·이메일) **부모만 입력**
- 결과 데이터는 부모 이메일에만 연결, 자녀 식별 정보 0
- 30일 후 자동 삭제 (성인용 90일보다 짧게)

## 5. UX Flow — 부모 게이트 (Parent Gate)

```
[랜딩: 부모 대상 카피]
     ↓
[부모 동의 화면]
  - "자녀의 만 나이를 선택하세요"
  - "본 테스트는 임상 진단이 아닙니다" 체크박스
  - "법정대리인으로서 자녀의 응시에 동의합니다" 체크박스
     ↓
[자녀 핸드오프 안내]
  - "이제 자녀에게 휴대폰을 건네주세요"
  - 한 번 더 disclaimer
     ↓
[자녀 응시: 20문항 / 5분]
  - 큰 글자, 큰 버튼, 격려 메시지
     ↓
[자녀 결과 화면: 격려 중심]
  - "잘했어요! 결과는 부모님께 보내드릴게요"
     ↓
[부모 결제: 이메일 입력]
     ↓
[부모 PDF 리포트: 상세 분석 + 가이드]
```

**핵심**: 결제 전 자녀 응시 완료 → 결제는 부모만. 부모가 결과 일부를 미리 보고(요약), 상세는 결제 후 PDF.

| 시나리오 | 가격 | 근거 |
|---|---|---|
| 보수적 | 4,900원 (성인용과 동일) | 가격 저항 최소 |

| 공격적 | 14,900원 (성인용 3배) | 부모용 학습 가이드 + 자녀 리포트 분리 등 가치 명시 필요 |

**MVP 권장**: 7,900원. 부모 결제 시장은 가격 민감도가 낮고, 4,900원은 오히려 "싸구려" 인상 줄 수 있음.

## 7. 결제·이메일 인프라

### 7.1 LS Store 분리 필요?
**Yes, 권장**: 별도 LS store 생성.
- 이유: 아동용 KYC 추가 검증을 성인용 store와 분리해서 한쪽 막힘이 다른 쪽 안 막게
- 추가 작업: 새 store registration, 새 API key/webhook 4세트

### 7.2 환경변수 추가
```
LEMON_SQUEEZY_KIDS_API_KEY=
LEMON_SQUEEZY_KIDS_STORE_ID=
LEMON_SQUEEZY_KIDS_PRODUCT_VARIANT_ID=
LEMON_SQUEEZY_KIDS_WEBHOOK_SECRET=
```

### 7.3 이메일
- Resend 동일 도메인 사용 가능, 발신자 이름만 "QuickIQ Kids" 등으로 차별화
- 템플릿 분리: 부모 대상 톤 (친근하지만 신뢰감)

## 8. 기술 구현 — 코드베이스 통합 vs 분리

**권장: 통합** (같은 Next.js 앱, 라우팅 분기)

```
app/[locale]/
├── page.tsx                 # 성인용 랜딩
├── kids/
│   ├── page.tsx             # 아동용 랜딩 (부모 대상)
│   ├── consent/             # 부모 동의
│   ├── test/                # 자녀 응시
│   └── result/[sessionId]/  # 결과
└── ...
```

**DB schema 추가**:
```sql
alter table test_sessions add column audience text default 'adult'; -- 'adult' | 'kids'
alter table test_sessions add column child_age int;                 -- nullable, 7-13
alter table test_sessions add column parent_consent_at timestamptz; -- 동의 시각
```

**문항 풀 분리**: `lib/questions/kids-ko.ts`, `lib/questions/kids-en.ts`
