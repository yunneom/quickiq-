# PRD — IQ Test MVP

## 1. 사용자 여정 (User Flow)

```
[인스타 스토리 광고]
       ↓ 스와이프업/링크 탭
[랜딩 페이지 /]
   - 후킹 카피 + CTA "지금 무료로 테스트"
   - 진입 직후 Meta Pixel "PageView" 발사
       ↓ 버튼 클릭
[테스트 시작 /test]
   - 익명 세션 생성 (Supabase anon)
   - 30문항, 한 번에 한 문항씩
   - 문항당 평균 15초, 총 7~8분 소요
   - 상단 진행률 바, 타이머 표시
   - "PageView" → "InitiateCheckout"(테스트 시작) 이벤트
       ↓ 마지막 문항 제출
[채점 + 결과 페이지 /result/{sessionId}]
   - 무료 결과: "당신의 추정 IQ는 상위 N%입니다"
   - 영역별 점수 4개 항목 흐릿하게 보여줌 (블러 처리)
   - CTA "9,900원으로 상세 리포트 받기"
       ↓ CTA 클릭
[체크아웃 /checkout/{sessionId}]
   - 이메일 입력 (1개 필드)
   - Lemon Squeezy 결제 위젯 (카카오페이/카드/Apple Pay 등)
   - "AddToCart" 이벤트
       ↓ 결제 완료
[Lemon Squeezy Webhook → 서버]
   - 결제 검증
   - PDF 자동 생성
   - Resend로 이메일 발송 (PDF 첨부)
   - Meta Conversion API로 "Purchase" 전송
       ↓
[감사 페이지 /thank-you]
   - "이메일을 확인하세요" 메시지
   - 결과 페이지 링크 (영구 잠금 해제)
```

## 2. 핵심 기능

### 2.1 랜딩 페이지
- **목적**: 광고 클릭 → 테스트 시작 전환율 최대화
- **요소**:
  - 헤드라인: "당신의 IQ는 상위 몇 %일까?" (한/영)
  - 신뢰성 신호: "심리측정학 기반 30문항", "전 세계 N만 명 응시"
  - CTA 버튼 1개 (피처폰 모니터에서도 명확히 보이게)
  - 면책 문구 (footer): "본 결과는 추정치이며 임상 진단이 아닙니다"
- **성능 요구**: LCP < 2.5s, CLS < 0.1, 인터랙티브 < 3s

### 2.2 테스트 진행
- **30문항**, 영역 4개 균등 분배:
  - 언어 추론 (Verbal Reasoning) 7-8문항
  - 수리 추론 (Numerical Reasoning) 7-8문항
  - 공간 추론 (Spatial Reasoning) 7-8문항
  - 논리 추론 (Logical Reasoning) 7-8문항
- **UX 규칙**:
  - 한 번에 한 문항만 표시 (모바일 집중도)
  - 뒤로가기 불가 (점수 신뢰성)
  - 문항당 60초 타임아웃 (자동 다음 문항, 오답 처리)
  - 진행률 바 + 남은 문항 수
  - 이탈 방지: 페이지 떠나려 하면 confirm 다이얼로그

### 2.3 점수 계산 로직
- **원점수**: 30문항 중 정답 개수
- **추정 IQ 변환**: 표준 정규분포 가정, 평균 100, 표준편차 15
  - 정답률을 백분위로 변환 후 IQ로 매핑
  - 예: 정답 24/30 (80%) → 상위 16% → 약 IQ 115
- **영역별 점수**: 각 영역 정답률 × 100
- **공식 문서**: `lib/scoring/` 안에 명확히 코멘트

### 2.4 무료 결과 (Result Page)
- **공개 정보**:
  - 종합 추정 IQ 백분위 ("상위 N%")
  - 영역별 점수 4개 (블러 처리, 클릭하면 결제 유도)
  - 평균 점수 비교 그래프 (블러)
- **CTA**: "전체 리포트 9,900원으로 받기"
- **공유 버튼**: 카카오톡/X/Facebook 공유 (바이럴 유도, "나는 상위 N%였어요")
  - 공유 시 본인 점수는 노출, 다른 사람은 자기 테스트로 유도

### 2.5 결제 (Lemon Squeezy)
- **단일 상품**: "Detailed IQ Report" — $7.50 / 9,900원
- **결제 흐름**:
  1. 사용자 이메일 입력
  2. 서버에서 Lemon Squeezy Checkout URL 생성 (custom_data에 sessionId 포함)
  3. 클라이언트가 해당 URL로 리다이렉트
  4. 결제 완료 → Webhook 수신
- **Webhook 처리** (`api/webhooks/lemon-squeezy`):
  - 서명 검증 (HMAC SHA256)
  - sessionId로 테스트 결과 조회
  - PDF 생성
  - Resend로 이메일 발송 (PDF 첨부)
  - Meta CAPI로 Purchase 이벤트 전송
  - DB에 결제 기록 저장
- **Idempotency**: Lemon Squeezy event_id를 unique key로 저장, 중복 처리 방지

### 2.6 상세 리포트 (PDF)
- **포함 내용**:
  - 종합 추정 IQ + 백분위
  - 영역별 4개 점수 + 강점/약점 분석
  - 동일 연령대 비교 그래프
  - 영역별 능력 해석 (각 200자 내외, 일반적/조심스러운 표현)
  - 응시일, 응시 ID
- **디자인**: A4 1-2장, 깔끔한 인포그래픽 스타일
- **언어**: 사용자가 테스트 본 언어로 발급 (ko/en)

### 2.7 i18n
- **지원 언어**: 한국어(기본), 영어
- **자동 감지**: `Accept-Language` 헤더 기반 첫 진입 시 결정, URL에 `/ko` `/en` 명시
- **모든 문자열**: `messages/{locale}.json`에 분리

## 3. 비기능 요구사항

### 3.1 성능
- 랜딩 LCP < 2.5s (광고 유입은 첫인상이 전부)
- 테스트 페이지 전환 < 200ms (문제 간 답답함 제거)
- PDF 생성 < 3s (Webhook 안에서 처리 가능한 수준)

### 3.2 모바일 최적화
- 모든 인풋 폰트 16px 이상 (iOS 자동 줌 방지)
- 터치 타깃 최소 44x44px
- 가로/세로 양쪽 대응
- 상단 노치 영역 safe-area-inset 처리

### 3.3 보안
- Webhook 서명 검증 필수
- Rate limiting: IP당 시간당 테스트 시작 10회
- SQL Injection 방지 (Supabase 쿼리 빌더만 사용, raw SQL 금지)
- 환경변수: 클라이언트 노출용은 `NEXT_PUBLIC_` 접두사만

### 3.4 컴플라이언스
- 개인정보처리방침 페이지 (이메일 수집 사실, 보관 기간 명시)
- 이용약관 페이지 (환불 정책 포함: "디지털 콘텐츠 특성상 발송 후 환불 불가")
- 면책 조항: "본 점수는 추정치이며 의학적/임상적 진단이 아닙니다"

## 4. 분석 이벤트 (Meta Pixel + CAPI)

| 이벤트명 | 발생 시점 | 값 |
|---------|---------|---|
| PageView | 랜딩 진입 | - |
| StartTest | 테스트 첫 문항 진입 | - |
| CompleteTest | 30문항 모두 응답 | - |
| ViewContent | 결과 페이지 진입 | - |
| InitiateCheckout | 결제 페이지 진입 | value: 9900 |
| Purchase | Webhook 결제 확인 | value: 9900, currency: KRW |

CAPI는 서버에서 직접 호출 (iOS14 픽셀 차단 우회).

## 5. Out of Scope (이번 주말 빌드에서 명시적 제외)

- 사용자 가입/로그인
- 결제 후 환불 처리 UI (수동 처리)
- 어드민 대시보드 (Supabase 대시보드로 충분)
- 푸시 알림
- 한국어/영어 외 언어
- A/B 테스트 인프라 (가격은 코드 수정으로 변경)
- 정기 결제/구독
- 친구 초대 리워드 시스템

## 6. 출시 후 첫 주 측정 지표

- 랜딩 → 테스트 시작 전환율 (목표 40%+)
- 테스트 시작 → 완료율 (목표 70%+)
- 결과 → 결제 전환율 (목표 3%+)
- CPA (광고 비용 / 결제 건수) — 9,900원 이하면 흑자
- 평균 테스트 소요 시간 (10분 이내가 이상적)
